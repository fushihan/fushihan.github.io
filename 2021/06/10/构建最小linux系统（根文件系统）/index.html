构建最小linux系统（根文件系统）

一、搭建环境和编译器配置

1. ubantu 16.04

sudo passwd root激活root用户

apt-get install vim ssh安装vim 和 ssh服务

vim /etc/ssh/sshd-config配置ssh服务，如果ssh不能使用的话，详情见《unbantu-16.4无法使用ssh.md》文档

安装镜像打包工具：

apt install android-tools-fsutils

apt-get install mkimage

2. 交叉编译器

在/usr/local/arm目录下解压安装arm交叉编译器tar -jxvf arm-XXX.tar.bz2

安装库文件：

apt-get install libncurses5-dev lib32ncurses5 lib32z1

二、busybox配置

解压busybox，在busybox目录下配置busybox

2. busybox的menuconfig配置

设置编译环境：“Build Options”栏找到“Cross compiler prefix”写入arm-none-linux-gnueabi-
设置安装目录：在“Installation Options ”栏找到“(./_install) Destination path for 'make install' (NEW)”写入../sysytem

选中setting中的vi-style line editing commands(NEW)

取消Linux Module Utilities中的Simplified modutils

使能支持中文：选中“Support Unicod”和“Check $LC_ALL, $LC_CTYPE and $LANG environment variables“

3. 安装根文件系统

在busybox目录下，使用make命令编译，再使用命令make install安装根文件系统。

7. 完善库文件

Busybox 编译生成的二进制文件是以动态链接库的形式运行，所以需要拷贝编译器里面的库文件到“lib”目录

编译器的库文件放在arm-xxx/arm-none-linux-gnueabi/libc/lib/目录下

cp /usr/local/arm/arm-2014.05/arm-none-linux-gnueabi/libc/lib/* ./

8. 打包和编译

make_ext4fs -s -l 314572800 -a -root -L linux system.img system

注：

制作ext4文件系统需要使用make_ext4fs命令，例如：

#make_ext4fs -s -l 314572800 -a root -L linux ./rootfs_qt.img ./root

执行之后即会将root文件打包成 rootfs_qt.img 文件系统镜像。

下载： make_ext4fs。
-————

*关于各参数的含义*

命令行下执行make_ext4fs即可看到使用方式：

root@DingMZ:~# make_ext4fs

Expected filename after options

make_ext4fs [ -l ] [ -j ] [ -b ]

[ -g ] [ -i ] [ -I ]

[ -L

[ -z | -s ] [ -J ]

[]

-——————————————————————

-s 就是生成ext4的S模式制作；

-l 314572800 是分区大小；

-a root 是指这个img用于Linux系统 （若为-a system即表示为android系统，挂载点即是/system。使用这个参数，make_ext4fs会根据private/android_filesystem_config.h里定义好的权限来给文件夹里的所有文件重新设置权限，如果你刷机以后发现有文件权限不对，可以手工修改android_filesystem_config.h来添加权限，重新编译make_ext4fs，也可以不使用 “-a system”参数，这样就会使用文件的默认权限 ）。

./rootfs_qt.img 表示在当前目录下生成镜像文件。

./root 指定源路径
